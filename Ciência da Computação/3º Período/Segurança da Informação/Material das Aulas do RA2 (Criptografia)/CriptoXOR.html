OPERAÇÕES BINÁRIAS EM CRIPTOGRAFIA: XOR


A) Operações lógicas em criptografia

O shift de caracteres usado pelo Caseser Cipher pode ser implementado como uma soma no código do caractere.

A soma é apenas um dos possíveis tipos de operações binárias que podem ser usadas na criptografia.

Entre as operações binárias, apenas aquelas que foram inversíveis (isto é, permitirem a volta ao formato original) são possíveis.

As operações lógicas tem a vantagem de não alteraro tamanho da palavra original, mas APENAS UMA DELAS É INVERSÍVEL.

O objetivo deste exercício é verificar quais das operações a seguir são inversíveis:

  • Complemento (NOT)  ~
  • E lógico (AND) &
  • OU lógico (OR) |
  • OU lógico eXclusivo (XOR) ^

Obs. NOT pode ser usado para negar o resultado de outras operações lógicas: NOT AND = NAND, NOT OR = NOR


Exercicio 1: Altere o operador usado na criptografia e verifique qual deles é inversível

def tabela_logica():
    dados = [ [ 0b0, 0b0 ], [ 0b0, 0b1 ], [ 0b1, 0b0 ], [ 0b1, 0b1 ] ]

    for x in dados:
       print(x, "AND", x[0] & x[1])
       print(x, "OR", x[0] | x[1])
       print(x, "XOR", x[0] ^ x[1])

def cripto(mensagem, chave):
    cripto = ""
    for c in mensagem:
        codigo = ord(c) + chave
        cripto += chr(codigo)
    return cripto

Exercicio 2: Altere o algoritmo para que a criptografia seja feita com rotação de bits

def ror(n, chave, tam):
    return (2**tam-1) & (n >> chave | n << (tam-chave) )

def rol(n, chave, tam):
    return (2**tam-1) & (n << chave | n >> (tam-chave) )
